# Justin Barnett

I build AI systems that have to work inside real tools: game editors, developer terminals, private data workflows, and messy human-in-the-loop operations.

The common thread is practical autonomy. I like systems that give AI enough context and tools to do useful work, while keeping humans in charge of judgment, safety, and taste.

## Built

### [MCP for Unity](https://github.com/CoplayDev/unity-mcp)

[![stars](https://img.shields.io/github/stars/CoplayDev/unity-mcp?style=flat&label=stars)](https://github.com/CoplayDev/unity-mcp)
[![forks](https://img.shields.io/github/forks/CoplayDev/unity-mcp?style=flat&label=forks)](https://github.com/CoplayDev/unity-mcp/forks)
[![license](https://img.shields.io/github/license/CoplayDev/unity-mcp?style=flat)](https://github.com/CoplayDev/unity-mcp/blob/main/LICENSE)

I built the original Unity MCP bridge that lets AI assistants control the Unity Editor through the Model Context Protocol. It gave Claude, Cursor, and other MCP clients tools for scenes, assets, scripts, materials, console logs, validation, and editor automation.

The project became a widely used open-source Unity AI tool and was sold to Coplay, where it continues as `CoplayDev/unity-mcp`.
